# coldturkey

NSFW blocker for this laptop, installed 2026-07-22. Turning it off requires
transcribing a random 80-character code shown on screen (Cold Turkey style).
Pasting is detected and rejected; the code is drawn with lookalike Unicode
glyphs, so a copied string never matches the expected plain-ASCII input.

## Layers

1. `/etc/hosts`: StevenBlack unified+porn list (~166k entries) between
   `>>> coldturkey blocklist >>>` markers, plus Google/Bing SafeSearch pins and
   the Firefox DoH canary (`use-application-dns.net`). The file is set
   immutable (`chattr +i`) while enforcement is on.
2. systemd-resolved drop-in (`/etc/systemd/resolved.conf.d/90-coldturkey.conf`):
   all lookups on every network go to Cloudflare family DNS (1.1.1.3), which
   also filters adult content server-side. Campus names (`*.stonybrook.edu`)
   still use the network's own DNS via the link search domain. Tailscale
   MagicDNS is unaffected.
3. Browser policies: DNS-over-HTTPS disabled and locked in Firefox (flatpak +
   host paths), Zen (`/etc/zen`), and Chromium/Chrome/Brave (managed policy
   JSON, which also forces Google SafeSearch). Zen and Chromium flatpaks get a
   read-only mount of their host policy dir via `flatpak override`.

4. Reddit NSFW filter (v1.1.0): DNS can't see URL paths, so NSFW subreddits on
   the otherwise-clean reddit.com are blocked in the browser policy layer
   instead: Firefox/Zen `WebsiteFilter` and Chromium `URLBlocklist`, generated
   from `/etc/coldturkey/reddit-nsfw.list` (syntax documented in the file:
   prefix stems, `name=` exact, `+name` exceptions; r/pornfree stays open)
   plus as many entries from the bulk list as fit under the browsers' 1000-
   pattern policy caps. Locked like the rest of the policies; covers
   old./new./np. subdomains.
5. Reddit uBlock list (v1.2.0): the full ~2k-name bulk list
   (`/etc/coldturkey/reddit-nsfw-bulk.list`) plus stems, `||redgifs.com^`
   (whole domain, all subdomains), and cosmetic rules that hide NSFW-flagged
   posts inline in feeds (`shreddit-post[nsfw]`, old reddit `.thing.over18`).
   Regenerated by enforce, served by `coldturkey-listserv.service` at
   http://127.0.0.1:8531/reddit.txt. ONE-TIME SETUP per browser: uBlock
   dashboard -> Filter lists -> Import -> paste that URL -> Apply. uBlock
   matching is case-insensitive, which closes the mixed-case hole the policy
   layer has. While blocking is off, the served list is empty but uBlock keeps
   its cached copy until its next update or a manual purge; reddit rules
   linger through short unlock windows (accepted; the rest really turns off).
   Extra hosts pins (redgifs CDN names) live in `/etc/coldturkey/extra-hosts`.
   Known remaining leaks: brand-new sub names, redd.it short links, reddit
   rehosting NSFW video on v.redd.it (the cosmetic rules hide flagged posts,
   and direct v.redd.it links to NSFW posts land on a blocked /r/ page).

Enforcement: `coldturkey-enforce.timer` (every 5 min + on boot) and
`coldturkey-watch.path` (fires on file tampering) run `coldturkey enforce`,
which reverts changes. `coldturkey-update.timer` refreshes the list weekly.
Enforce runs are serialized by a lock (v1.2.0) and verify that resolved is
actually serving the family DNS, restarting it when the drop-in isn't live.

## Commands

    coldturkey                    status
    sudo coldturkey off           challenge, then off for 1h (--for 30m/2h/1d, --forever)
    sudo coldturkey on            re-enable now, no challenge
    sudo coldturkey allow X.com   challenge, then allowlist a false positive
    sudo coldturkey disallow X.com
    sudo coldturkey update        refresh blocklist now
    coldturkey demo               try the challenge, changes nothing
    sudo coldturkey uninstall     challenge, then full removal

## Notes

- Restart browsers once after install so the DoH policy loads. Check
  `about:policies` in Firefox/Zen or `chrome://policy` in Chromium.
- To edit `/etc/hosts` yourself: `sudo coldturkey off` first (the immutable
  flag blocks edits), make the change outside the markers, `sudo coldturkey on`.
- Config knobs in `/etc/coldturkey/config.json`: `challenge_length` (floor 40),
  `display_style: "plain"` if the styled glyphs render as boxes,
  `youtube_restricted: true` to force YouTube Restricted Mode,
  `safesearch`, `dns_layer`, `browser_policies`, `lock_hosts_immutable`.
- The allowlist only affects the hosts layer. If Cloudflare family DNS blocks
  a site server-side, there is no per-site override; switching the DNS layer
  to a NextDNS profile would add that.
- Honest limits: you have root, so with enough deliberate steps you can rip
  this out. The point is that no single impulsive action gets through: hosts
  edits bounce off `chattr +i`, deleted files come back within seconds, DNS
  stays filtered on every network, and the sanctioned exit is typing 80
  characters by hand. VPN apps that set their own DNS bypass layer 2 (the
  hosts layer still applies). Other devices are not covered.
- Backups: originals of every file this touched are in
  `/var/lib/coldturkey/backup/`, tracked by `/var/lib/coldturkey/files.json`.
